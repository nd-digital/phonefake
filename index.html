📱 PhoneFake — Guide

Cet outil simule un appareil mobile (iPhone ou Android) dans ton navigateur, en portrait ou paysage. Il sert à tester l'apparence et le comportement de tes propres applications web — qu'elles soient classiques, responsive, PWA installables ou autres.

Aperçu : 5 formats de simulation — iPhone, Android, iPad, Tablette Android et Ordinateur — interchangeables d'un clic.

✨ Fonctionnalités

  • 2 modèles d'appareil : iPhone (notch + coins arrondis) ou Android (punch-hole + coins plus carrés).
  • Rotation : bascule portrait ↔ paysage en un clic, l'écran et les décorations s'adaptent.
  • Comparaison : le bouton Compare affiche jusqu'à 3 appareils côte à côte (téléphone, tablette, ordinateur). Chaque colonne a son sélecteur ; l'option Désactivé en masque un (au moins deux restent visibles).
  • Agrandissement : le bouton ⛶ à côté de chaque écran l'affiche en grand dans une fenêtre (pleine hauteur du viewport) ; ⤡ ou Échap revient à l'état initial.
  • Zoom : chaque appareil a un curseur de zoom vertical (à droite, quand une appli est ouverte). Glisse pour agrandir ; une fois zoomé, glisse dans la fenêtre pour te déplacer (comme sur un téléphone) et reviens à 100 % pour cliquer.
  • Synchro entre écrans : en comparaison, tes actions dans l'appli (navigation, défilement, clics, saisie) sont répercutées sur les autres écrans. Automatique si l'appli est servie sur la même origine que PhoneFake ; sinon, ajoute une ligne dans ton appli : <script src="…/phonefake-sync.js"></script>.
  • Clavier virtuel : quand un champ de ton appli reçoit le focus, un clavier monte et réduit le viewport (comme un vrai téléphone) — pour vérifier que tes champs restent visibles et que la mise en page tient. (Nécessite l'agent de synchro ci-dessus.)
  • Multi-applis : place tes propres applis dans des sous-dossiers à côté de cette page (chacune avec son index.html) et elles apparaissent dans le menu Mode web.
  • Apps « serveur » (Node/pm2) : une app qui tourne sur son propre port (ex. localhost:3002), sans index.html ? Ajoute un phonefake.json à sa racine — { "url": "http://localhost:3002" } — sinon tu obtiens un listing « Index of… ». Clés en option : name, icon.
  • Compatible : sites classiques, responsive design, PWA installables, applis avec service worker — tout ce qui se sert dans un navigateur.
  • Accueil et Applis : reviens à un launcher minimal ou retrouve la liste des applis ouvertes.
  • Multilingue : interface en 5 langues (🇫🇷 français, 🇬🇧 anglais, 🇪🇸 espagnol, 🇮🇹 italien, 🇩🇪 allemand), sélectionnable via les drapeaux en haut.
  • Mode desktop : le sélecteur Mode web en haut de la page permet aussi d'ouvrir une appli dans un nouvel onglet du navigateur, en pleine largeur (rendu desktop sans simulation).
  • Tableau de bord : le bouton ↩ index localhost (en haut à gauche) renvoie à INDEX_LARAGON, un tableau de bord open-source qui liste tous tes projets locaux. S'il n'est pas installé, une page t'invite à le faire (aucune dépendance : PhoneFake marche seul).

💡 Utilisation

  • Ajoute tes applis dans des sous-dossiers du même répertoire que ce fichier ; elles apparaissent automatiquement dans le sélecteur.
  • Choisis iPhone ou Android pour adapter le rendu de ton appli aux contraintes de chaque plateforme.
  • Utilise Rotation pour vérifier l'affichage paysage de ton interface.

🖥️ Usage local uniquement

PhoneFake est conçu pour un environnement de développement local (Laragon, MAMP, XAMPP, serveur Node, etc.). Ce n'est pas un service en ligne — il sert à tester rapidement tes propres applications sur ta machine, avant déploiement.

⌨️ Raccourcis clavier

  • 1 iPhone · 2 Android · 3 iPad · 4 Tablette · 5 Ordinateur
  • R Rotation · C Comparer · H Aide · A Accessibilité · Échap Fermer

♿ Accessibilité

Tes choix sont mémorisés et s'appliquent uniquement à cette page.

＋ Nouvelle appli

Entre un nom : PhoneFake crée un dossier avec une structure PWA de base (point d'entrée, manifeste, service worker, page hors-ligne, css/js) et l'affiche aussitôt.

🔒 Auteur et compte GitHub sont enregistrés uniquement en local sur ta machine (un fichier ignoré par git + ton navigateur). Ces infos ne quittent jamais l'appli et ne sont jamais envoyées sur GitHub.
